## Codebase Patterns
- Python 3.13 available via `python3` (not `python`) on this machine
- Use `python3 src/collector.py` to run the main entry point
- All source code in `src/` as a Python package with `__init__.py`
- Config loaded from `config.json` at project root
- Use `dataclasses` for data models (Tweet, etc.)
- aiohttp used for async HTTP downloads (media_downloader.py)
- Twitter tweet data path: `data.home.home_timeline_urt.instructions[].entries[].content.itemContent.tweet_results.result`
- `TweetWithVisibilityResults` is a wrapper typename â€” unwrap via `.tweet` to get the real tweet result
- Retweets have `legacy.retweeted_status_result.result` containing the original tweet
- Promoted tweets have `itemContent.promotedMetadata` in the entry
- Cursor entries have `itemType: "TimelineCursor"` â€” skip these when parsing tweets
- `extended_entities.media` has all media; `entities.media` only has the first â€” prefer extended_entities
- Use `source .venv/bin/activate` before running python commands (venv required due to externally-managed Python)
- Use `wait_until="domcontentloaded"` for Twitter page loads â€” `networkidle` may never fire due to constant streaming
- Twitter login redirect check: look for both `/login` and `/i/flow/login` in URL
- Validate session JSON before passing to Playwright to catch corrupted files early
- `sys.path.insert(0, ...)` needed in entry-point scripts that use `from src.` imports when run as `python3 src/file.py`
- Interceptor uses `page.on("response", handler)` â€” handler must be async since it calls `await response.json()`
- Reload page after attaching interceptor to capture initial timeline GraphQL requests
